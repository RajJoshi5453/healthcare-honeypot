# Healthcare IoT Deception Honeypot Network

> **Cybersecurity Internship Project** — Simulate vulnerable medical IoT devices to trap, log, and analyze attackers before they reach real hospital infrastructure.

## What This Project Does

This repository builds a complete, containerized honeypot network that emulates a **BusyBox-based medical HVAC controller** (`MedDevice-HVAC-01`) inside a hospital network. Attackers who scan and brute-force the device are granted a fake shell, and every keystroke, login attempt, command, and uploaded file is captured for analysis.

## Stack & Tools

| Layer | Tool | Purpose |
|-------|------|---------|
| Honeypot Engine | Cowrie | SSH / Telnet emulation & session capture |
| Container Runtime | Docker + Compose | Isolation, reproducibility, easy reset |
| Log Storage | Elasticsearch | Indexing and search of structured attack events |
| Visualization | Kibana | SOC dashboards, histograms, timelines |
| ETL Pipeline | Logstash | Real-time JSON log ingestion & normalization |
| Log Parser | Python 3.10 | Offline aggregation, IoC extraction |
| GeoIP Lookup | ip-api.com | Free, keyless geolocation enrichment |
| Dashboard | Flask + Chart.js | Custom SOC-style UI for internship presentation |
| Map | Leaflet.js | World map with attacker origin pins |

## Quick Start

### 1. Prerequisites

- Docker + Docker Compose
- Python 3.10+ (for offline parsers)

### 2. Clone & Configure

```bash
git clone <repo-url> healthcare-honeypot
cd healthcare-honeypot
cp .env.example .env
# Edit .env if your paths differ (defaults work for most layouts)
```

### 3. Generate the Fake Filesystem

Cowrie needs a `fs.pickle` binary that defines the fake medical IoT filesystem.

```bash
cd honeypot/cowrie
python3 generate_fs.py
cd ../..
```

### 4. Deploy the Stack

```bash
cd infrastructure
docker compose up --build -d
```

Services spin up on the isolated `honeypot-isolated` bridge network:
- **Cowrie SSH** on port `2222`
- **Cowrie Telnet** on port `2223`
- **Kibana** on `http://localhost:5601`
- **Flask Dashboard** on `http://localhost:5000`
- **Elasticsearch** on `http://localhost:9200` (internal + host)

### 5. Ingest & Analyze Logs (after capturing some attacks)

```bash
cd log-parser
python3 -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt

# Parse raw Cowrie JSON logs
python3 parser.py

# Extract IoCs, deduplicate IPs, command frequency, top credentials
python3 ioc_extractor.py

# Enrich attacker IPs with country, city, lat/lon via ip-api.com
python3 geoip_enricher.py
```

### 6. View the Dashboard

Open [http://localhost:5000](http://localhost:5000) to see:
- Real-time attack stats (total sessions, unique IPs, commands, uploads)
- Top 5 commands bar chart (Chart.js)
- Most targeted credentials
- Links to the **World Map** and **Top 20 IoCs** table

## Key Security Features

1. **Zero Hardcoded Secrets** — All paths and hosts are loaded from `.env` via `python-dotenv`.
2. **Docker Network Isolation** — Honeypot containers run on a dedicated bridge (`172.25.0.0/24`). They cannot reach the hospital LAN unless explicitly routed.
3. **Read-Only Malware Mounts** — Captured samples are analyzed in isolated VMs; the dashboard mounts `analysis/` as read-only.
4. **Rate-Limited GeoIP** — `geoip_enricher.py` sleeps 1 second between requests to respect `ip-api.com` free-tier limits.

## Reset & Cleanup

```bash
./honeypot/scripts/reset_honeypot.sh
```

This destroys containers, prunes volumes, and wipes captured logs/malware samples. **Back up `analysis/attack_patterns.json` first if you need historical data.**
